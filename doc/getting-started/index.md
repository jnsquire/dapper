# Getting Started

This section collects the primary onboarding paths for Dapper, from first-time setup through more advanced debugging workflows.

- [Installation](installation.md) - install Dapper and verify that the CLI is available
- [Quick Start](quickstart.md) - launch the adapter and run a first debugging session
- [Using Dapper with VS Code](using-vscode.md) - configure and use Dapper from Visual Studio Code
- [Standalone Adapter Setup](standalone-adapter.md) - run Dapper outside the VS Code extension flow
- [Manual Testing](manual-testing.md) - exercise the adapter and launcher directly during development
- [Async Debugging](../guides/async-debugging.md) - work with asyncio tasks, async-aware stepping, and concurrency scenarios
- [Frame Evaluation](../guides/frame-eval.md) - enable lower-overhead debugging and troubleshoot frame-eval behavior
- [Variable Presentation](../guides/variable-presentation.md) - understand how Dapper renders complex Python objects
- [Hot Reload](../guides/hot-reload.md) - reload edited modules while paused in a debugging session
